# WoW Server Assistant (WSA)

`WSA` — Windows-приложение для управления WoW сервером (AzerothCore/TrinityCore):  
`authserver`, `worldserver` и службой `MySQL/MariaDB` из одного окна.

## Что публикуется в этом репозитории

Здесь размещаются **готовые релизные сборки**:
- `WSA-Setup-<version>.exe` (установщик)

Исходники и инструкции по разработке в этом README не требуются.

## Основные возможности

- Отдельный `Start / Reset / Stop` для:
  - `MySQL`
  - `All` (Auth + World)
  - `Auth`
  - `World`
- Цветовые индикаторы состояния:
  - красный: остановлен
  - желтый: запуск/остановка
  - зеленый: работает
- Логи в реальном времени:
  - вкладки `Auth` и `World`
  - подсветка warning/error
  - копирование выделения `Ctrl + C`
  - кнопка `Copy log`
- Отправка команд в серверную консоль (`Send`)
- Экран `Settings`:
  - пути к `authserver/worldserver`
  - аргументы запуска
  - `ReadyRegex`
  - имя службы MySQL
  - расписание `Regular Reboot Time`
  - запуск с Windows
- Поддержка языков: `RU / EN`

## Системные требования

- Windows 10/11 (x64)
- Права администратора (нужны для управления службой MySQL)

## Установка

1. Скачайте `WSA-Setup-<version>.exe` из раздела **Releases**.
2. Запустите установщик и завершите установку.
3. Откройте `WoW Server Assistant`.

## Быстрый старт

1. Откройте `Settings`.
2. Укажите `Build folder` или вручную пути к:
   - `authserver.exe`
   - `worldserver.exe`
3. Проверьте аргументы:
   - обычно `-c authserver.conf`
   - обычно `-c worldserver.conf`
4. При необходимости задайте `ReadyRegex` (можно оставить пустым — используются встроенные значения).
5. Нажмите `Save`.
6. Запускайте сервисы кнопками слева.

## Где хранятся файлы

- Конфиг: `config.json` рядом с `WSA.exe`
- Логи лаунчера: `Logs/launcher-YYYYMMDD.log`

## Частые проблемы

### Порт занят (`Could not bind to 0.0.0.0:3724`)
Уже запущен другой `authserver/worldserver` или порт используется другим процессом.

### Ошибка доступа к MySQL (`Access denied`)
Запустите `WSA` от имени администратора.

### Сервер не переходит в `Running`
Проверьте:
- правильность путей к exe
- аргументы `-c ...`
- корректность `ReadyRegex` (или оставьте пустым)
